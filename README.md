## Overview

This project implements a production-ready LLM inference API with:
- **Dynamic request batching** — groups concurrent requests to amortise GPU compute
- **In-process caching** — stores responses with TTL expiry and LRU eviction
- **Comprehensive benchmarks** — latency, throughput, and cache-hit-rate measurements
- **Privacy-preserving design** — cache keys are SHA-256 hashes; no user IDs stored

## Quick Start (under 5 minutes)

### 1. Clone and install dependencies

```bash
git clone https://github.com/[your-username]/ids568-milestone5-[netid].git
cd ids568-milestone5-[netid]
pip install -r requirements.txt
```

### 2. Start the inference server

```bash
python -m src.server
```

The server starts on `http://localhost:8000`. You should see:

```
INFO  Server ready on 0.0.0.0:8000
```

### 3. Test a request

```bash
curl -X POST http://localhost:8000/generate \
     -H "Content-Type: application/json" \
     -d '{"prompt": "What is machine learning?"}'
```

Expected response:
```json
{"text": "[Model response to: ...]", "cached": false, "latency_ms": 105.3}
```

### 4. Run benchmarks

In a **separate terminal** (server must be running):

```bash
python benchmarks/run_benchmarks.py
```

Results are saved to `benchmarks/results/benchmark_results.json` and charts to `analysis/visualizations/`.

## Configuration

All settings are configurable via environment variables:

| Variable           | Default | Description                                       |
|--------------------|---------|---------------------------------------------------|
| `BATCH_SIZE`       | `8`     | Max requests per batch                            |
| `BATCH_TIMEOUT_MS` | `50`    | Max wait (ms) before dispatching incomplete batch |
| `CACHE_TTL_SECONDS`| `300`   | Seconds before a cache entry expires              |
| `CACHE_MAX_ENTRIES`| `1000`  | Max entries before LRU eviction                   |
| `MODEL_NAME`       | `mock`  | Set to HuggingFace model ID for real inference    |
| `PORT`             | `8000`  | Server port                                       |
| `LOG_LEVEL`        | `INFO`  | Logging verbosity (DEBUG, INFO, WARNING)          |

### Example: Custom configuration

```bash
BATCH_SIZE=16 BATCH_TIMEOUT_MS=100 CACHE_TTL_SECONDS=60 python -m src.server
```

### Using a real LLM

```bash
MODEL_NAME=gpt2 python -m src.server
```

> **Note:** Requires `torch` and `transformers` (uncomment in `requirements.txt`).

## API Reference

### `POST /generate`

Run inference.

**Request body:**
```json
{
  "prompt": "string",
  "max_new_tokens": 50,
  "temperature": 0.7,
  "use_cache": true
}
```

**Response:**
```json
{
  "text": "string",
  "cached": false,
  "latency_ms": 104.2
}
```

### `GET /health`

Liveness check. Returns `{"status": "ok"}`.

### `GET /stats`

Real-time metrics for cache and batcher.

### `POST /cache/clear`

Flush the response cache (useful for cold-cache benchmarks).

## Benchmark Scenarios

| Scenario                     | What it measures                                  |
|------------------------------|---------------------------------------------------|
| Cold vs Warm cache           | Latency improvement from cache hits               |
| Load levels (low/med/high)   | Throughput (req/s) at 10 / 50 / 100 req/s        |
| Serial vs Concurrent         | Per-request latency with and without batching     |
| Cache hit-rate over time     | How quickly the cache warms up                    |

Run with `--base-url` if using a non-default address:

```bash
python benchmarks/run_benchmarks.py --base-url http://my-server:8000
```
